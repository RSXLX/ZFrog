## 二、产品愿景与目标

### 2.1 产品定位升级

| 维度     | 旧版 ZetaFrog        | 新版 ZetaFrog 2.0                |
| -------- | -------------------- | -------------------------------- |
| 核心玩法 | 跨链旅行 + AI日记    | **养成责任 + 跨链旅行 + AI日记** |
| 用户参与 | 被动等待青蛙旅行归来 | **主动照顾 + 被动旅行双循环**    |
| 情感连接 | 弱（青蛙只是工具）   | **强（青蛙是需要照顾的生命）**   |
| 玩家留存 | 依赖旅行奖励         | **依赖养成责任感 + 旅行奖励**    |

### 2.2 核心用户价值

1. **情感寄托**：养育一个有生命周期的数字伙伴

2. **成就感**：看着青蛙从蛋孵化、成长、进化、繁衍

3. **探索乐趣**：通过良好的照顾解锁更丰富的跨链探索

4. **社交连接**：与好友互动、配对繁殖、建立链上关系网

## 三、用户角色与场景

### 3.1 用户画像

| 用户类型   | 特征                 | 核心需求                     | 使用频率     |
| ---------- | -------------------- | ---------------------------- | ------------ |
| 佛系玩家   | 追求陪伴感，不喜欢肝 | 简单照顾、AI对话、欣赏日记   | 每天5-10分钟 |
| 养成玩家   | 喜欢培育、收集       | 进化分支、繁殖收集、完美养成 | 每天30分钟+  |
| 社交玩家   | 喜欢互动、炫耀       | 好友互动、配种服务、排行榜   | 每天多次短时 |
| GameFi玩家 | 追求收益             | NFT交易、任务奖励、代币增值  | 按任务周期   |

### 3.2 核心用户旅程

```plaintext
新用户流程：
连接钱包 → Mint青蛙蛋 → 孵化等待(互动加速) → 青蛙诞生 → 命名
    ↓
日常养成循环：
查看状态 → 喂食/清洁/玩耍 → 状态恢复 → 获得奖励 → 解锁旅行
    ↓
旅行循环：
准备干粮 → 发起跨链旅行 → 等待归来 → 查看日记/纪念品 → 分享
    ↓
社交循环：
添加好友 → 串门互动 → 提升亲密度 → 配对繁殖 → 新生命诞生
```

#### B.2 清洁功能

**触发机制**：

- 随机事件：每2-4小时触发"排泄事件"，清洁度-15~25

- 事件发生时：青蛙旁边出现💩图标，环境变脏

**操作流程**：

```plaintext
检测到排泄事件 → 显示脏乱状态 → 用户点击"清洁"按钮 
    → 播放清扫动画 → 清洁度重置100 → 获得少量$LILY奖励
```

**长期不清洁后果**：

- 清洁度<40：青蛙表情变差，幸福度衰减加速

- 清洁度<20：健康度开始衰减

- 清洁度=0：青蛙生病

#### B.3 玩耍功能

**功能描述**：与青蛙互动，提升幸福度，获得$LILY奖励

**小游戏列表**：

| 游戏名   | 解锁条件 | 玩法                   | 奖励                   |
| -------- | -------- | ---------------------- | ---------------------- |
| 猜方向   | 初始解锁 | 猜青蛙跳向哪边         | 幸福度+10, 10-30$LILY  |
| 接虫子   | Lv.3     | 控制青蛙接住掉落的虫子 | 幸福度+15, 20-50$LILY  |
| 跳荷叶   | Lv.5     | 跳跃躲避障碍           | 幸福度+20, 30-80$LILY  |
| 记忆翻牌 | Lv.8     | 配对相同图案           | 幸福度+15, 40-100$LILY |
| 节奏敲击 | Lv.12    | 按节奏点击             | 幸福度+25, 50-150$LILY |
| 链上问答 | Lv.15    | 回答区块链知识         | 幸福度+30, XP+50       |

**每日限制**：每个游戏每天最多玩5次

#### B.4 治疗功能

#### B.5 休息功能

**作息规则**：

- 游戏时间与现实时间同步

- 22:00-06:00 为休息时段

- 休息期间活力值自动恢复+30

- 休息期间其他属性衰减速度减半

**手动休息**：

- 用户可点击"灯光"按钮让青蛙提前休息

- 休息时青蛙显示💤动画

- 休息中的青蛙无法操作

#### E.2 繁殖系统详细设计

**繁殖条件**：

- 双方青蛙均为成熟期

- 双方亲密度达到100

- 双方玩家均确认同意

- 支付繁殖费用: 5 ZETA (双方各付一半)

- 双方青蛙均不在冷却期

**繁殖流程**：

```plaintext
发起配对请求 → 对方确认 → 支付费用 → 链上合约执行
    → 生成子代基因 → Mint青蛙蛋NFT → 分配蛋的归属
    → 双方进入7天繁殖冷却期
```

**遗传规则**：

| 属性类型       | 遗传方式                               |
| -------------- | -------------------------------------- |
| 外观(皮肤颜色) | 70%随机继承父/母，30%混合变异          |
| 性格           | 50%继承父，30%继承母，20%随机变异      |

#### H.1 死亡条件

| -------- | ---------------------- | ------ |

- 进入死亡倒计时时，每6小时发送一次紧急提醒

- 最后6小时内每小时提醒

- 最后1小时内持续提醒

#### H.2 死亡处理流程

```plaintext
死亡判定 → 链上记录死亡事件 → 生成墓碑NFT → 展示"一生回顾"
    ↓
继承结算:
├── 50% $LILY 存入"遗产池"
├── 部分道具可继承
├── 家园装饰保留
└── 好友关系记录保留
    ↓
重新开始选项:
├── 免费领取新蛋 (继承部分遗产)
└── 购买特殊蛋 (更好的初始属性)
```

- 青蛙名字、生卒日期

- AI生成的"墓志铭"
