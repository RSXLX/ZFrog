# ZetaFrog UX 优化需求设计

> 基于 ui-ux-pro-max 技能进行系统性 UX 分析和优化设计

---

## 一、设计系统定义

### 1.1 产品定位

- **类型**: Web3 宠物养成 + 游戏化社交
- **风格**: Playful, Gamification, Cute, NFT
- **技术栈**: React + TypeScript + Tailwind CSS + Framer Motion

### 1.2 设计风格

**主风格: Claymorphism (黏土风格)**

- 柔和阴影与多层光影
- 圆润元素 (rounded-2xl ~ rounded-3xl)
- 流畅过渡动画 (150-300ms)
- 渐变色与微妙质感

**辅助风格: Glassmorphism**

- 用于弹窗、悬浮面板
- `backdrop-blur-xl` + 半透明背景

### 1.3 色彩系统

| 用途       | 主色                | 辅助色      | 状态色                 |
| ---------- | ------------------- | ----------- | ---------------------- |
| 主题       | Green-500 (#22c55e) | Emerald-400 | -                      |
| 养成/饥饿  | Orange-400          | Amber-500   | 警告: <30%, 危险: <15% |
| 养成/快乐  | Pink-400            | Rose-500    | 警告: <40%, 危险: <20% |
| 养成/清洁  | Cyan-400            | Teal-500    | 警告: <30%, 危险: <15% |
| 养成/健康  | Red-400             | Rose-500    | 警告: <30%, 危险: <15% |
| 养成/活力  | Yellow-400          | Amber-500   | 警告: <30%, 危险: <15% |
| 跨链旅行   | Purple-500          | Indigo-600  | -                      |
| $LILY 货币 | Yellow-400          | Orange-400  | -                      |

### 1.4 字体系统

- **标题**: Orbitron (科技感)
- **正文**: Exo / Inter (可读性)
- **代码/地址**: font-mono

| 问题 ID | 问题描述                                                  | 严重程度 | 影响页面     |
| ------- | --------------------------------------------------------- | -------- | ------------ |
| NAV-01  | **没有全局导航栏**，用户在子页面难以快速切换              | 🔴 高    | 全局         |
| NAV-02  | Home 页面的 Command Center 入口只在右侧展示，移动端体验差 | 🟡 中    | Home         |
| NAV-03  | FrogDetail 页面顶部按钮太多 (5 个)，信息过载              | 🟡 中    | FrogDetail   |

## 四、优化方案设计

### 4.1 Phase 1: 核心体验优化 (必做)

#### 4.1.1 全局导航系统

**新增组件**: `GlobalNavbar.tsx`

```
┌────────────────────────────────────────────────────────┐
│ 🐸 ZetaFrog     [Home] [My Frog] [Journal] [Friends]     [Wallet] │
└────────────────────────────────────────────────────────┘
```

- 固定顶部导航栏
- 响应式: 移动端 → 底部 Tab Bar
- 当前页面高亮指示
- 钱包连接状态常驻显示

**实现要点**:

1. App.tsx 中统一添加 GlobalNavbar
2. 底部 Tab Bar 使用 `position: fixed; bottom: 0`
3. 页面内容添加 `padding-bottom` 避免遮挡

#### 4.1.2 统一 Toast 通知系统

**新增组件**: `ToastProvider.tsx` + `useToast.ts`

替换所有 `alert()` 调用，统一使用:

```typescript
const { toast } = useToast();
toast.success("喂食成功！青蛙吃饱了~");
toast.error("余额不足");
toast.info("旅行已开始");
```

**设计规范**:

- 位置: 右下角堆叠
- 动画: slide-in from right
- 自动消失: 3s

#### 4.1.3 状态实时更新指示

**优化组件**: `StatusPanel.tsx`

1. 添加 "上次更新" 时间戳
2. 添加 "刷新" 按钮带 loading 动画
3. 状态变化时添加微动画 (数字滚动)

```
┌─────────────────────────────────────┐
│ 🍖 饥饿度  ████████████░░ 82%      │
│            上次更新: 2分钟前 [🔄]    │
└─────────────────────────────────────┘
```

#### 4.1.4 替换 Emoji 图标

**全局替换规则**:

| Emoji | 替换为                     | 图标库       |
| ----- | -------------------------- | ------------ |
| 🐸    | Lucide `Frog` 或自定义 SVG | -            |
| 🏠    | Lucide `Home`              | lucide-react |
| 🎁    | Lucide `Gift`              | lucide-react |
| 🔄    | Lucide `RefreshCw`         | lucide-react |
| ✈️    | Lucide `Plane`             | lucide-react |
| 👥    | Lucide `Users`             | lucide-react |

**例外**: 表情类 emoji (如日记心情 😊🤩) 可保留

---

### 4.2 Phase 2: 流程优化 (推荐)

#### 4.2.1 养成/旅行 Tab 切换优化

**当前问题**:

- mainTab 在 FrogDetail 中切换 'nurture' / 'travel'
- 旅行中时隐藏 Tab 选择器

**优化方案**:

- 添加 Tab 切换动画 (slide left/right)
- 旅行中时 Tab 保留但 disabled，显示 "青蛙正在旅行中..."
- 养成面板添加 "快速出发" 入口

#### 4.2.2 旅行状态可视化增强

**优化组件**: `TravelStatus.tsx`

添加状态时间线可视化:

```
[LOCKED] ─────► [ON_TARGET_CHAIN] ─────► [CROSSING_BACK] ─────► [UNLOCKED]
   ✓                  ●                       ○                    ○
```

#### 4.2.3 小游戏解锁提示

**优化组件**: `NurturePanel.tsx`

未解锁游戏显示锁定状态 + 解锁条件:

```
┌────────────────┐
│  🔒 接虫子      │
│  Lv.3 解锁     │
│  (当前 Lv.2)   │
└────────────────┘
```

#### 4.3.2 Haptic Feedback
